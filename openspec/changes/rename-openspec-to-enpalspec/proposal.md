## Why

enpalspec is a fork of openspec, but the CLI binary, skill templates, and command files still reference `openspec` in skill names, CLI invocations, and program identity. This creates a confusing experience where users see `openspec` in help output, skill folder names, and skill content even though the tool is `enpalspec`.

## What Changes

- All skill template `name:` fields renamed from `openspec-*` to `enpalspec-*`
- All `dirName` values in skill generation renamed from `openspec-*` to `enpalspec-*`
- `SKILL_NAMES` array in tool detection updated to `enpalspec-*`
- `WORKFLOW_TO_SKILL_DIR` mappings in `init.ts` and `profile-sync-drift.ts` updated
- All CLI invocations inside skill templates and command files changed from `openspec` to `enpalspec`
- Cross-references to `openspec-sync-specs` and `openspec-continue-change` updated to `enpalspec-*`
- Default `compatibility` and `author` metadata in skill generation updated
- `.name('openspec')` in CLI program updated to `.name('enpalspec')`
- User-facing strings in `src/cli/index.ts` updated
- `.claude/commands/enpalspec/*.md` and `.claude/commands/opsx/*.md` CLI calls updated

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `enpalspec-binary`: CLI program name (`.name()`) changes from `openspec` to `enpalspec`, affecting help text and error messages.

## Impact

- Installed skills (generated by `enpalspec init`) will be placed in `enpalspec-*` directories instead of `openspec-*`
- Users who already ran `enpalspec init` will have stale `openspec-*` skill folders; they need to re-run `enpalspec init` to get the renamed skills
- `legacy-cleanup.ts` detection patterns (`openspec-*.md` globs) are intentionally left unchanged — they exist to clean up files installed by the upstream openspec tool
- `openspec/` data directory on disk is not renamed
- `bin/openspec.js` filename is not renamed (internal only; binary is already exposed as `enpalspec` via `package.json`)
